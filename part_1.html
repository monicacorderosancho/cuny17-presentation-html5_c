Introducción

HyperText Markup Language HTML, Cascading Style Sheets CSS, y JavaScript JS son los principales lenguajes que hacen que la web funcione.

Estos lenguajes están estrechamente relacionados, pero están diseñados para tareas muy específicas. Vamos a comprender cómo interactúan durante esta sesión.

HTML es para agregar significado a tu contenido, dandole marcado (markup).

CSS es para darle formato (estilo) al contenido marcado.

JavaScript es para darle interacción al formato y el contenido marcado.

¿Cómo funciona cada uno?

HTML

Puedes marcar una ejecución particular de texto como un párrafo con este HTML:

Este es un párrafo.

CSS

Luego, puedes especificar el tamaño y color del párrafo con algo de CSS:

p {
    font-size: 20px;
    color: blue;
}

JavaScript

Y si nos queremos poner exquisitos, podemos re-escribir el párrafo cuando el usuario clickea, con algo de JavaScript

var p = document.getElementById('parrafo');

    p.addEventListener('click', function(event) {
      p.innerHTML = 'Hiciste click en este párrafo.';
    });

Conclusión

HTML, CSS y JavaScript son lenguajes totalmente diferentes.

La mayoría de los sitios web dependen de los tres, pero la apariencia de cada sitio está determinada por HTML y CSS.

Comencemos creando una nueva carpeta en nuestra computadora, llamemosla ejercicios.
Luego, dentro de la carpeta, crehtmlfccaremos un nuevo archivo HTML llamado 01.html.

Este archivo HTML representa una página web, y es donde pondremos todo nuestro código.

El flujo de trabajo básico hoy será editar HTML en el editor de texto y luego ver los cambios en un navegador web.

Estructura de una página web

Agreguemos el siguiente código HTML a nuestro archivo 01.html.

<!DOCTYPE html>
    <html lang="es">
        <head>
        <title>Título de mi página</title>
        </head>
        <body>
            <h1>Armas de distracción masiva</h1>
            <h2>La agnotología es el estudio de la fabricación premeditada de desconocimiento</h2>
            <p>Se aproximan las elecciones francesas y alemanas y, en uno y otro país, políticos y periodistas se preguntan...</p>
        </body>
    </html>
<!-- Declaración de tipo de documento -->
<!DOCTYPE html>
    <!-- Etiqueta raíz con declaración de lenguaje -->
    <html lang="es">
        <!-- Metadata de la página -->
        <head>
        <!-- Titulo de la página -->
        <title>Armas de distracción masiva</title>
        <!-- Cierre de la etiqueta HEAD -->
        </head>
        <!-- Etiqueta de texto encabezado principal -->
        <body>
            <!-- Etiqueta de texto encabezado principal -->
            <h1>Armas de distracción masiva</h1>
            <!-- Etiqueta de texto encabezado secundario -->
            <h2>La agnotología es el estudio de la fabricación premeditada de desconocimiento</h2>
            <!-- Etiqueta de texto de párrafo -->
            <p>Se aproximan las elecciones francesas y alemanas y, en uno y otro país, políticos y periodistas se preguntan...</p>
        <!-- Cierre de la etiqueta BODY -->
        </body>
    <!-- Cierre de la etiqueta principal HTML -->
    </html>

Veamos que és cada parte

Primero, necesitamos decir a los navegadores que se trata de una página web HTML5 con la línea :

<!DOCTYPE html>

Esta es sólo una cadena especial que los navegadores buscan cuando intentan mostrar nuestra página web.

Declaración de tipo de documento en HTML 4.01

<!DOCTYPE html PUBLIC
        "-//W3C//DTD XHTML 1.0 Transitional//EN"
        "http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd">

Elementos de bloque y elementos dentro de línea

Todos los elementos HTML tienen un valor por default para visualizarse, dependiendo en el tipo de contenido que sea.

Elementos de bloque (block)

Un elemento de nivel de bloque siempre comienza en una nueva línea y ocupa todo el ancho disponible (se extiende hacia la izquierda y la derecha hasta donde puede).

Block Elements

  • <header>
  • <footer>
  • <section>
  • <nav>
  • <article>
  • <aside>
  • <h1–h6>
  • <div>
  • <p>
  • <pre>
  • <figcaption>
  • <figure>

  • <ol>
  • <ul>
  • <li>
  • <table>
  • <tbody>
  • <tr>
  • <td>

Elementos dentro de línea (inline)

Un elemento en línea no se inicia en una nueva línea y sólo ocupa tanto ancho como sea necesario.

Inline Elements

Listas

HTML contiene 3 tipos de elementos para generar listas.

Listas ordinales (ordered list)
  • <ol>
Listas no ordinales (unordered list)
  • <ul>
Listas de descripción (description list)
  • <dl>

Al mismo tiempo, existen 3 etiquetas complementarias

Items de lsita
  • <li>
Item de definición de término
  • <dt>
Item de descripción de definición
  • <dd>

Listas ordinales (ordered list)

  • <ol>
  • </ol>
Etiqueta tipo
Contenido de Texto
Función
Declara una lista ordenada

Ejemplo:

<ol type="A">
  <li>Ciudad de México</li>
  <li>São Paulo</li>
  <li>Buenos Aires</li>
</ol>

Listas no ordinales (unordered list)

  • <ol>
  • </ol>
Etiqueta tipo
Contenido de Texto
Función
Declara una lista no ordenada

Ejemplo:

<ol type="circle">
  <li>Theresa May</li>
  <li>David Cameron</li>
  <li>Gordon Brown</li>
</ol>

Item de lista

  • <li>
  • </li>
Etiqueta tipo
Contenido de Texto
Función
Declara un elemento de lista, compatible con listas ordenadas y no ordenadas

Ejemplo:

<ol type="circle">
  <li>Piedra</li>
  <li>Papel</li>
  <li>Tijera</li>
</ol>

Listas de descripción (description list)

  • <dl>
  • </dl>
Etiqueta tipo
Contenido de Texto
Función
Declara una lista de términos, con una descripción para cada término.

Ejemplo:

<dl>
  <dt>Nombre:</dt>
  <dd>John Connor</dd>
</dl>
